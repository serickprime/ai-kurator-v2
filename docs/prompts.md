# Prompts

Prompt design must enforce the evidence-first contract.

## Answer Prompt Requirements

- Include the user question.
- Include only `QuestionAnalysis`, `EvidencePack`, the user question, and compact dialog context for follow-up questions.
- Do not include raw candidate chunks, document candidates, retrieval candidates, discarded candidates, or reranker rejects.
- Tell the model to answer only from evidence.
- Tell the model to say when evidence is insufficient.
- Ask for concise, friendly Russian text.
- Avoid decorative Markdown in Telegram answers.

Prompt changes are part of the RAG contract. Do not edit prompts to hide routing or retrieval failures; fix the earliest failing stage and add an eval case for the pattern.

## Answer Generation Prompt

```text
Ты отвечаешь в evidence-first RAG pipeline.

Входные данные ограничены:
- QuestionAnalysis;
- EvidencePack;
- исходный вопрос пользователя;
- краткий контекст диалога, если это follow-up.

Правила:
- Используй только EvidencePack.
- Raw candidates, retrieval candidates и discarded candidates запрещены и не должны учитываться.
- Не добавляй строку "По материалам" вручную.
- Не выдумывай точные команды, SQL, API-параметры, настройки нод, версии, сроки курса или ссылки без evidence.
- Не добавляй источники вручную: sources строит приложение после проверки.
- Если evidence не хватает, коротко скажи, чего именно нет.
- Пиши кратко, как куратор: дружелюбно, делово, без декоративного markdown.
```

### Answer Modes

- `answer_from_materials`: answer strictly from `evidence_items`, cover `must_answer_points`, and say which points are not covered.
- `partial_answer`: say that materials contain partial information, answer only the confirmed part, and ask for missing data.
- `ask_for_missing_data`: do not diagnose; ask briefly for the specific missing input.
- `general_answer_without_sources`: give a general answer without the phrase "По материалам" and without source lines.
- `out_of_base`: say that the materials do not contain confirmed evidence for the question; do not include source lines.

## Claim Verification Prompt Requirements

- Include the answer draft.
- Include the same evidence pack.
- Ask which claims are supported, unsupported, or missing evidence.
- Return JSON:

```json
{
  "verdict": "pass|rewrite|fail",
  "unsupported_claims": [],
  "safe_answer": "",
  "source_leakage": false
}
```

## Source Prompt Requirements

Sources should not be generated by the model from memory.

The application derives sources only from `evidence_pack.source_matches` after generation and verification.

Sources must not be derived from raw candidates, document candidates, discarded candidates, or partial evidence that is not marked as a source.
